# Бизнес-эффект детектора фейковых отзывов

## Методология расчета

### Допущения

1. **Стоимость фейкового отзыва**: 500 руб.
   - Средняя стоимость возврата товара
   - Потеря доверия покупателя
   - Репутационные издержки

2. **Объем данных**: На основе собранных отзывов продавца

3. **Точность модели**: Используем метрики Precision и Recall

## Расчет экономии

### Формула

```
Экономия = TP × Стоимость_фейка
```

Где:
- **TP (True Positives)** — количество правильно обнаруженных фейков
- **Стоимость_фейка** — 500 руб.

### Пример расчета

Предположим:
- Всего отзывов: 500
- Фейковых отзывов (по разметке): 150 (30%)
- Модель обнаружила: 120 фейков
- True Positives: 100
- False Positives: 20

**Экономия:**
```
100 × 500 руб. = 50,000 руб.
```

**Потери от ложных срабатываний:**
```
20 × 200 руб. (стоимость проверки) = 4,000 руб.
```

**Чистая экономия:**
```
50,000 - 4,000 = 46,000 руб.
```

## Масштабирование

### На уровне маркетплейса

Если применить модель ко всем продавцам:

- Средний продавец: 500 отзывов/месяц
- Количество продавцов: 100,000
- Доля фейков: 30%

**Потенциальная экономия:**
```
100,000 продавцов × 500 отзывов × 0.3 × 500 руб. × Precision
= 7,500,000,000 руб./месяц (при Precision = 0.8)
= 6,000,000,000 руб./месяц
```

## ROI модели

### Затраты на разработку

- Разработка: 200 часов × 3,000 руб./час = 600,000 руб.
- Инфраструктура: 50,000 руб./месяц
- Поддержка: 100,000 руб./месяц

**Итого в первый год:** 2,400,000 руб.

### Доход

При внедрении на 1,000 продавцов:
```
1,000 × 500 отзывов × 0.3 × 500 руб. × 0.8 = 60,000,000 руб./месяц
```

**ROI:**
```
(60,000,000 × 12 - 2,400,000) / 2,400,000 × 100% = 29,900%
```

2. **Защита честных продавцов**
   - Снижение недобросовестной конкуренции

3. **Юридические риски**
   - Снижение жалоб от покупателей
   - Соответствие законодательству о защите прав потребителей
